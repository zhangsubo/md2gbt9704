# GB/T 9704-2012 Markdown to DOCX 转换器

## 标准规格

| 项目 | 规格 |
|------|------|
| 纸张 | A4 |
| 上边距 | 37mm |
| 下边距 | 35mm |
| 左边距 | 27mm |
| 右边距 | 27mm |
| 行间距 | 固定值 28 磅 |
| 每页行数 | 可视情况增删 1-2 行 |

## 字体规范

| 元素 | 字体 | 字号 | 加粗 | 对齐 |
|------|------|------|------|------|
| 主标题 | 方正小标宋简体 | 2号（22pt） | 否 | 居中 |
| 一级标题（一、） | 黑体 | 3号（16pt） | 是 | 左对齐 |
| 二级标题（（一）） | 楷体_GB2312 / 楷体 | 3号（16pt） | 是 | 左对齐 |
| 三级标题（1.） | 仿宋_GB2312 / 仿宋 | 3号（16pt） | 是 | 左对齐 |
| 四级标题（（1）） | 仿宋_GB2312 / 仿宋 | 3号（16pt） | 否 | 左对齐 |
| 正文 | 仿宋_GB2312 / 仿宋 | 3号（16pt） | 否 | 两端对齐 |
| 页码 | Times New Roman | 4号（14pt） | 否 | 单右双左 |

## 标题层级结构

正文层次依次使用：
1. **一、** → 一级标题，黑体加粗
2. **（一）** → 二级标题，楷体加粗
3. **1.** → 三级标题，仿宋加粗
4. **（1）** → 四级标题，仿宋不加粗

## 排版规则

- 标题与正文之间空一行
- 所有段落（包括标题和正文）首行缩进 2 字符（约 9pt）
- 页码：单页居右，双页居左，格式为 `— 页码 —`

## 使用方式

```bash
python3 <skill_path>/scripts/md2docx.py <输入文件.md> <输出文件.docx>
```

## Markdown 标题映射规则

| Markdown 语法 | 识别为 | 格式 |
|--------------|--------|------|
| `# 标题` | 主标题 | 方正小标宋，22pt，居中 |
| `## 一、xxx` 或 `一、` | 一级标题 | 黑体加粗，16pt |
| `### （一）xxx` 或 `（一）` | 二级标题 | 楷体加粗，16pt |
| `#### 1. xxx` 或 `1.` | 三级标题 | 仿宋加粗，16pt |
| `##### （1）xxx` 或 `（1）` | 四级标题 | 仿宋不加粗，16pt |
| `**文本**` | 加粗文本 | 仅在标题中生效，正文中忽略 |

### 推荐 Markdown 写法

```markdown
# 公文标题

## 一、总则

本文档用于说明相关事项。

### （一）总体目标

本年度工作以数字化转型为核心。

### （二）基本原则

1. 坚持党建引领
2. 坚持创新驱动

## 二、重点任务

### 1. 推进数字化建设

#### （1）信息系统升级

加快现有信息系统的升级改造，确保数据安全。

#### （2）数据治理

建立完善的数据治理体系。
```

## 字体跨平台说明

脚本会自动适配系统可用字体：

| 优先级 | Windows | Mac | Linux |
|--------|---------|-----|-------|
| 主标题 | 方正小标宋简体 | 方正小标宋简体 | 尝试检测，无则回退 |
| 仿宋 | 仿宋_GB2312 | 仿宋 | 尝试检测 |
| 黑体 | 黑体 | STHeiti / 黑体 | 尝试检测 |
| 楷体 | 楷体_GB2312 | 楷体 | 尝试检测 |

**注意**：Linux 系统若无中文字体，Word 打开时会回退到默认字体。

## 依赖

- `python-docx`：未安装时运行 `pip install python-docx`
- 中文字体：系统需安装方正小标宋简体、黑体、楷体、仿宋字体

## 故障排除

- **字体显示为默认字体**：系统缺少对应字体，Word 打开时会自动回退
- **行距不符合预期**：可调整脚本中的 `LINE_SPACING` 变量（推荐 26-30 范围）
- **页码显示问题**：在 Word/WPS 中打开后，如需更新页码，请选择"更新域"
