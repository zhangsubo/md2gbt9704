# md2gbt9704

将 Markdown 文档按照 [GB/T 9704-2012（党政机关公文格式）](https://openstd.samr.gov.cn/bzgk/gb/newGbInfo?hcno=F3CC9BEF482524C895FDA7A08BB4A70E)标准转换为 DOCX 格式。

## 功能

- 按照党政机关公文格式标准生成 Word 文档
- 自动识别 Markdown 标题层级并映射为公文标题格式
- 支持 Markdown 表格转换（表头加粗，0.5磅边框）
- 自动处理落款格式（单位名称和日期右对齐）
- 自动去除 Markdown 列表标记（`*`、`+`、`-`）
- 正文中不出现加粗（符合公文标准）
- 支持跨平台字体适配（Windows / macOS / Linux）
- 自动生成符合标准的页码格式

## 标准规格

| 项目 | 规格 |
|------|------|
| 纸张 | A4 |
| 上边距 | 37mm |
| 下边距 | 35mm |
| 左边距 | 27mm |
| 右边距 | 27mm |
| 行间距 | 固定值 28 磅 |

## 字体规范

| 元素 | 字体 | 字号 | 加粗 | 对齐 |
|------|------|------|------|------|
| 主标题 | 方正小标宋简体 | 2号（22pt） | 否 | 居中 |
| 一级标题（一、） | 黑体 | 3号（16pt） | 是 | 左对齐 |
| 二级标题（（一）） | 楷体 | 3号（16pt） | 是 | 左对齐 |
| 三级标题（1.） | 仿宋 | 3号（16pt） | 是 | 左对齐 |
| 四级标题（（1）） | 仿宋 | 3号（16pt） | 否 | 左对齐 |
| 正文 | 仿宋 | 3号（16pt） | 否 | 两端对齐 |
| 表头 | 仿宋 | 小四号（12pt） | 是 | 居中 |
| 表格内容 | 仿宋 | 小四号（12pt） | 否 | 左对齐 |
| 落款 | 仿宋 | 3号（16pt） | 否 | 右对齐 |
| 页码 | Times New Roman | 4号（14pt） | 否 | 单右双左 |

## 安装

### 依赖

- Python 3.6+
- python-docx

```bash
pip install python-docx
```

### 字体

系统需安装以下中文字体（缺少时会自动回退）：

| 字体 | Windows | macOS | Linux |
|------|---------|-------|-------|
| 主标题 | 方正小标宋简体 | 方正小标宋简体 | 自动检测 |
| 仿宋 | 仿宋_GB2312 | 仿宋 | 自动检测 |
| 黑体 | 黑体 | STHeiti | 自动检测 |
| 楷体 | 楷体_GB2312 | 楷体 | 自动检测 |

## 使用方法

### 命令行

```bash
python3 scripts/md2docx.py <输入文件.md> [输出文件.docx]
```

如不指定输出文件，自动生成 `原文件名_format.docx`

示例：

```bash
# 自动生成 输出_format.docx
python3 scripts/md2docx.py 公文.md

# 指定输出文件名
python3 scripts/md2docx.py 公文.md 公文_output.docx
```

### 作为智能体 Skill 使用

本项目可作为 [opencode](https://opencode.ai) 或其他兼容 OpenClaw 协议的智能体的 Skill 使用。

#### 安装 Skill

- 生成公文格式文档

## Markdown 写法

### 标题映射

| Markdown 语法 | 识别为 | 格式 |
|--------------|--------|------|
| `# 标题` | 主标题 | 方正小标宋，22pt，居中 |
| `一、xxx` | 一级标题 | 黑体加粗，16pt |
| `（一）xxx` | 二级标题 | 楷体加粗，16pt |
| `1. xxx` | 三级标题 | 仿宋加粗，16pt |
| `（1）xxx` | 四级标题 | 仿宋不加粗，16pt |
| `* 文本` | 列表项 | 去除标记，作为正文 |
| `**文本**` | 加粗标记 | 正文中去除，标题中保留 |

也支持带 Markdown 标记的标题（如 `## 一、`、`### （一）`）。

### 表格格式

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据1 | 数据2 | 数据3 |
```

- 表头：仿宋加粗，小四号（12pt）
- 内容：仿宋不加粗，小四号（12pt）
- 边框：0.5磅直线

### 落款格式

```markdown
某某单位

2026年1月3日
```

- 单位名称（以公司/局/委等结尾）：右对齐，正文下空两行
- 日期（阿拉伯数字，年月日）：右对齐，单位名称下空一行

### 推荐模板

```markdown
# 关于印发《2026年工作要点》的通知

## 一、总则

根据上级文件精神，结合本单位实际，制定本方案。

### （一）总体目标

以数字化转型为核心，全面推进各项工作。

### （二）基本原则

1. 坚持党建引领
2. 坚持创新驱动

## 二、重点任务

### 1. 推进数字化建设

#### （1）信息系统升级

加快现有信息系统的升级改造工作，确保数据安全。

#### （2）数据治理

建立完善的数据治理体系，确保数据质量。

## 三、保障措施

| 措施 | 责任部门 | 完成时间 |
|------|----------|----------|
| 加强领导 | 办公室 | 1月 |
| 落实资金 | 财务处 | 2月 |

某某单位

2026年1月3日
```

## 页码格式

页码格式为 `— 1 —`（带破折号边框线）：
- 奇数页：居右
- 偶数页：居左

## 故障排除

| 问题 | 解决方案 |
|------|----------|
| 字体显示为默认字体 | 系统缺少对应字体，Word 打开时会自动回退，需安装对应字体 |
| 行距不符合预期 | 可调整 `scripts/md2docx.py` 中的 `LINE_SPACING` 变量（推荐 26-30 范围） |
| 页码显示问题 | 在 Word/WPS 中打开后，选择"更新域"刷新页码 |
